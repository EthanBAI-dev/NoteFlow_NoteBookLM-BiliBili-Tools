# NoteFlow — 产品介绍

> 让 NotebookLM 吃掉互联网上的一切知识。

## 一句话介绍

**NoteFlow** 是一款 Chrome 扩展，解决 Google NotebookLM "喂不进去"的问题——帮你把哔哩哔哩、YouTube、播客、网页、AI 对话等各种来源的内容一键导入 NotebookLM。

---

## 🎯 解决什么问题？

NotebookLM 是 Google 推出的 AI 笔记工具，能对你上传的资料进行深度分析、问答和生成播客。但导入来源太麻烦了：

- 想从哔哩哔哩导入视频字幕？手动复制粘贴
- YouTube 播放列表有 20 个视频？一个一个来
- 你有 5 个浏览器标签页想导入？手动复制每个 URL
- 和 Claude 讨论了一下午技术方案？对话被困在聊天窗口里
- 导入进度不知道，到底成功了没有？

**NoteFlow 就是来解决这些问题的。**

---

## ✨ 核心功能

### 🎬 视频字幕导入

**场景**：你在哔哩哔哩看到一个技术教程，想把字幕和弹幕导入 NotebookLM 做笔记。

**怎么用**：
1. 打开 B 站视频页面
2. 扩展自动识别，切换到「哔哩哔哩」标签
3. 选择要导入的字幕（支持多 P 视频）
4. 一键导入到 NotebookLM

**YouTube 同样支持**：粘贴视频/播放列表/频道链接，自动提取视频列表，勾选批量导入，可限制最近 N 个视频。

---

### 🎙️ 播客内容导入

**场景**：你在小宇宙听到一期两小时的播客，想快速获取核心内容。

**怎么用**：
1. 粘贴小宇宙播客链接
2. 扩展自动获取节目信息和单集列表
3. 选择要导入的单集
4. 一键导入到 NotebookLM，让 AI 生成摘要

---

### 🌐 网页内容导入

**场景**：你打开了 5 个网页想做研究，想全部导入 NotebookLM。

**怎么用**：
1. 切换到「网页」标签
2. 扩展自动列出当前浏览器所有窗口的标签页（按窗口分组）
3. 勾选要导入的网页（支持全选/取消全选）
4. 点击底部全局导入按钮，后台批量写入

**当前网页默认选中**，切换标签页时 SourceInfoCard 自动更新。

---

### 💬 AI 对话导入

**场景**：你和 Claude 讨论了一个技术方案，想把这段对话作为知识来源存入 NotebookLM。

**怎么用**：
1. 在 Claude / ChatGPT / Gemini 的对话页面
2. 打开扩展（自动定位到「AI 对话」标签）
3. 点击「提取当前对话」
4. 以问答对为单位，勾选要导入的内容
5. 一键导入到当前打开的 NotebookLM 笔记本

**支持三大平台**：Claude (claude.ai)、ChatGPT (chatgpt.com)、Google Gemini (gemini.google.com)。

---

### ⚡ 后台批量导入

选中多条内容后点击底部全局导入按钮，后台**并行**发送导入请求，所有 URL 同时写入 NotebookLM。

进度实时显示：
- `导入中... (1/5)` → `(2/5)` → ... → `(5/5)`
- 完成后显示 `导入成功 (5/5)`
- 失败时显示具体错误原因

无需打开新标签页，无需手动切换。

---

### 🔑 多账号与笔记本管理

- **Google 账号管理**：在导航栏左侧显示当前登录账号，支持下拉切换
- **Notebook 选择器**：在内容区顶部自由选择目标笔记本，导入内容自动发送到选中笔记本
- **自动检测**：自动识别当前 NotebookLM 页面的激活账号

---

### 📋 导入历史

自动记录最近 100 条导入记录，随时点击导航栏历史按钮回溯。

---

## 🏗️ 技术特点

- **纯客户端运行**：无服务器、无账号、无数据上传，内容处理全在浏览器内完成
- **Chrome MV3**：使用最新 Manifest V3 规范，符合 Chrome 安全要求
- **WXT + React + TypeScript**：现代化技术栈，代码可维护性强
- **并行导入**：通过 `Promise.allSettled` 并行发送导入请求，大幅缩短等待时间
- **智能 DOM 自动化**：深度适配 NotebookLM 的 Angular Material UI
- **i18n 国际化**：中英双语界面，自动适配浏览器语言

## 🎬 使用场景

### 场景 1：技术学习
> 我在 B 站看了一个 Vue3 教程（18 集），想把所有字幕导入 NotebookLM 做笔记。

→ 哔哩哔哩标签：勾选所有 P 的字幕 → 一键导入。

### 场景 2：YouTube 研究
> 我想把某个 YouTube 频道最近 10 个视频都导入 NotebookLM。

→ YouTube 标签：粘贴频道链接 → 限制 10 个 → 全选 → 导入。

### 场景 3：信息聚合
> 我打开了 8 个网页做竞品分析，想全部导入。

→ 网页标签：8 个标签页排好了 → 全选 → 导入。后台并行写入，不用等。

### 场景 4：知识沉淀
> 和 ChatGPT 讨论了一下午的方案，想把对话存进 NotebookLM。

→ AI 对话标签：打开 ChatGPT 页面 → 提取 → 勾选有价值的部分 → 导入。

### 场景 5：播客笔记
> 在小宇宙听了一期播客，想用 AI 生成摘要。

→ 播客标签：粘贴链接 → 导入 → NotebookLM 自动生成笔记。

## 💡 设计哲学

1. **降低门槛**：NotebookLM 最大的价值在于"喂进足够多的好内容"，但导入门槛太高会让人放弃。NoteFlow 让这件事变简单。
2. **批量优先**：一次性处理多条内容，而不是一次一条。网页标签支持全选，YouTube 支持播放列表/频道，后台并行处理。
3. **所见即所得**：SourceInfoCard 显示当前页面信息，进度条实时更新，成功/失败一目了然。
4. **纯本地运行**：用户的数据不离开浏览器，不需要信任任何第三方服务。
