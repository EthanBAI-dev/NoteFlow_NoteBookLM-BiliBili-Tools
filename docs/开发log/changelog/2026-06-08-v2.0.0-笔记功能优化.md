# 更新日志: v2.0.0 — 笔记触发逻辑优化 + 同目标聚合 + 全局开发日志

## 变更概要

全面优化笔记系统的触发逻辑、组织结构，新增全局开发日志模块和引擎实现，包含 20 个自动化测试用例。

## 变更内容

### 🎯 笔记触发逻辑优化

- **新增任务状态判断**：自动触发前检查任务是否已完成，未完成时切换场景不生成笔记
- **新增主动触发入口**：用户可通过 `记录一下` / `记个笔记` 或 `/record` 命令手动发起，不受任务状态限制
- **触发后流程重排**：先创建笔记 → 记录全局日志 → 再切换到新目标

### 📂 笔记组织结构优化

- **同目标聚合**：同一目标下可关联多条笔记，首次归档 + 后续改进迭代追加
- **内容去重机制**：基于 `computeDedupHash` 内容指纹检查，重复内容自动拦截并提示

### 📊 全局开发日志模块

- 新增 `DevLog-Global.md` 极简日志
- 首次完成归档时自动记录（改进迭代不重复）
- 包含：目标名称、完成时间（ISO）、完成摘要

### 🧩 引擎实现

- **新增文件** `lib/devlog-engine.ts` — DevLog 引擎核心类
  - `SessionState` 管理：currentGoalId、taskStatus
  - `switchGoal()` 触发决策：返回 allowAutoTrigger
  - `createNote()` 笔记创建：支持自动/手动模式
  - `isDuplicate()` / `findDuplicates()` 去重检查
  - `getGlobalLog()` / `generateGlobalLogMarkdown()` 全局日志
  - `exportData()` / `importData()` 序列化支持

### ✅ 测试验证

- **新增文件** `tests/devlog-engine.test.ts` — 7 组 20 个测试用例
  - 触发逻辑：任务状态判断（4 个）
  - 主动触发：手动入口验证（3 个）
  - 同目标聚合：笔记追加与不同目标隔离（2 个）
  - 去重校验：重复内容拦截（3 个）
  - 全局开发日志：自动记录与 Markdown 生成（4 个）
  - 序列化与恢复（1 个）
  - 工具函数（2 个）
